# データ分析AIエージェント向け プロンプト例集

---

## 🔴 売上・業績分析

### ❌ 悪い例

```
売上を分析して
```

**問題点**: 期間・比較対象・分解軸がすべて不明。AIは汎用的な集計しか返せない。

### ✅ 良い例

```
今年1〜3月の売上を、昨年同期と比較して商品カテゴリ別に見せて。
特に落ちているカテゴリがあれば原因も教えて。
```

**理由**: 期間・比較対象・分析軸・知りたいことが揃っている。

---

### ❌ 悪い例

```
先月の売上が悪かった理由を教えて
```

**問題点**: 「悪い」の基準・何と比べて悪いのかが伝わらない。

### ✅ 良い例

```
先月の売上が前月比で15%落ちた。
店舗別・曜日別・時間帯別に分解して、どこで何が起きているか教えて。
```

**理由**: 変化量を明示し、分解軸を指定することでAIが的外れな分析を避けられる。

---

## 🟠 異常・品質検知

### ❌ 悪い例

```
データに異常があれば教えて
```

**問題点**: 異常の定義・正常値がない。AIは統計的外れ値を機械的に返すだけになる。

### ✅ 良い例

```
不良品率は通常2%以下のはずなのに、今週の第3ラインで急に上がっている気がする。
いつから、どのくらい上がっているか確認して。
```

**理由**: 正常値・場所・時期というドメイン知識を渡すことで精度が上がる。

---

## 🟡 顧客・行動分析

### ❌ 悪い例

```
顧客を分析して
```

**問題点**: 「顧客分析」は切り口が無数にあり、方向性が決まらない。

### ✅ 良い例

```
去年まで毎月買ってくれていたのに今年は購入が止まった顧客を抽出して。
購入金額が大きかった順に並べて、どんな商品を買っていたか教えて。
```

**理由**: セグメント条件を業務言語で定義し、優先順位まで指定している。

---

## 🟢 予測・将来予測

### ❌ 悪い例

```
来月の売上を予測して
```

**問題点**: キャンペーン・季節性・外部要因などの業務背景が抜けている。

### ✅ 良い例

```
来月は例年より気温が高いとの予報で、去年の猛暑の月と傾向が似ている。
同じ条件の月のデータを参考に、来月の飲料カテゴリの売上を予測して。
```

**理由**: 外部要因と参照すべきデータをAIに渡すことで予測精度が上がる。

---

## 🔵 比較・ランキング

### ❌ 悪い例

```
どの店舗が良いか教えて
```

**問題点**: 「良い」の指標が不明。売上？利益率？客数？指標次第で答えが変わる。

### ✅ 良い例

```
今期、売上は高いのに利益率が低い店舗を見つけたい。
売上上位10店舗の利益率も一緒に並べて、傾向に違いがあれば教えて。
```

**理由**: 評価指標を2軸で明示し、問題意識を共有することでAIが本質に近づける。

---

## 💡 良いプロンプトの共通ポイント

| 要素 | ❌ 悪い例 | ✅ 良い例 |
|------|----------|----------|
| 期間 | なし | 「先月」「今年1〜3月」と明示 |
| 比較対象 | なし | 「前月比」「昨年同期比」を指定 |
| 正常の基準 | なし | 「通常は○○のはず」と伝える |
| 分解軸 | なし | 「店舗別・曜日別」など指定 |
| 問題意識 | なし | 「○○が気になっている」と共有 |
